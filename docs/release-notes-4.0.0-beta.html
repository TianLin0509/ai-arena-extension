AI Arena v4.0.0-beta Phase 2 完成

2026-05-22 · 群聊 popup 增强为独立完整工具，sidepanel 保留 + bug 修复

新增 — popup 群聊增强

  • 左下角参与者多选 roster：头像 toggle 选择下轮参与者，默认全选，状态持久化（chrome.storage.local），全空自动回弹防呆
  • 任务模式选择器：输入框左侧 ⚙️ 任务 按钮，hover 弹三级菜单 — 同时提问 / 辩论 (自由 / 群策) / 总结 (N 候选裁判) / PPT 制作
  • 紧凑 meta 行：每条气泡上方一行 = 名字 + 时间戳（SF Mono 等宽）+ 状态点（已完成/提取中/失败，含脉动动画）+ hover 浮现的操作按钮
  • 气泡 4 个操作按钮：🔄 重提（手动 readResponse）/ 📤 重发（重新 inject 同问题）/ 📋 复制 / ↗ 跳原页
  • 品牌 logo：用户头像 = 华为红圆（HW），9 个 AI 各用自己品牌 SVG（Claude 八角形 / Gemini 钻石 / OpenAI 莲花 / DeepSeek / 豆包 / 千问 / Kimi / 元宝 / Grok）
  • 视觉精致化：新消息 slide-in 动画、headers/bubbles backdrop-blur 毛玻璃、send 按钮弹性回弹 cubic-bezier、letter-spacing -0.01em 字距收紧

修复

  • sidepanel 自动提取失败 bug：旧检测条件依赖 streaming indicator (stop button) 的 DOM selector，selector 失效后 sidepanel 永远不会自动判定完成，需用户手动点提取。现简化为 popup 同款"N 次稳定文本"判定，不再依赖 streaming indicator。src/sidepanel.js:685

等价性保证（mirror）

popup 工具栏每个按钮的 background handler 与 sidepanel 完全一致：

  • 同时提问 → chatBroadcast
  • 辩论·自由/群策 → debateRound (style: free | collab)
  • 总结 → summary (judgeId)
  • PPT 工坊 → 当前提示用户在 sidepanel 完成（prompt builders 高度依赖 sidepanel 内部状态，未来重构后两边共用）
  • 气泡 重提/重发 → chatReextractOne / sendPromptToService

已知限制（v4.1+ 解决）

  • PPT 工坊三个 prompt 构造函数尚未抽到共享文件，popup 点 PPT 任务弹提示让用户在 sidepanel 完成
  • 历史回填（拉取 AI 原页已有 turns）仍未做（v1.1 计划）
  • Artifact / Mermaid / 图片仍跳原页不内嵌渲染（v3+ 计划）

体验建议

  • 启用并列模式（3 个 AI 各独立 window）+ 群聊 popup 独立 window，所有窗口都是 top-level，不踩 Chrome 后台 tab throttling
  • 双屏：副屏铺 AI 窗口，主屏右半放群聊 popup
  • 任务模式选择器选定后会持续生效（直到再切换），不像 @mention 用完即过
